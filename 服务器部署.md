# 内网服务器部署说明（Windows）

本文说明将本 OCR 微服务拷贝到**无外网**的 Windows Server 2016 / Windows 10 / Windows 11 服务器后，如何安装依赖、启动服务，以及配置**开机自动启动**。

> **说明（Windows）**：`gunicorn` 在 Windows 上不可用，生产与测试请使用项目根目录下的 `startup.bat`，其内部使用 `uvicorn` 多进程承载并发，与 Linux 下 “多 worker” 的思路一致。

---

## 1. 需要拷贝到服务器的内容

建议**整体拷贝**本仓库（或发布目录），至少包含下列路径（相对项目根目录）：

| 路径 | 用途 |
|------|------|
| `app/` | 服务代码 |
| `gunicorn_conf.py` | 仅作 worker 数等参数参考（Linux 可用）；Windows 以环境变量 + `startup.bat` 为准 |
| `requirements.txt` | 依赖声明 |
| `offline_bundle/models/` | PaddleOCR 检测/识别/方向模型（程序默认相对路径读取） |
| `offline_bundle/paddle/` | `paddlepaddle` CPU 及传递依赖的离线 wheel |
| `offline_bundle/wheels/` | 业务依赖离线包（含 `requirements-lock.txt` 锁定清单） |
| `安装程序/` | Python 安装包、`VC++` 运行库等（若服务器尚未安装） |
| `startup.bat` | 本机启动脚本 |
| `服务器部署.md` | 本文档 |
| `CSharp-IIS-调用示例.md` | （可选）C# 调用参考 |

**不必拷贝**（可在服务器上重新生成）：

- `.venv/`：在服务器上创建虚拟环境即可  
- `.venv_offline_build/`：仅用于外网打包依赖，可不带

**不要依赖**打包机上的 `D:\Python313\Lib\site-packages`。服务器安装应**只使用** `offline_bundle` 中的 wheel，并以 `requirements-lock.txt` 为准。

---

## 2. 服务器环境准备

1. 安装 **Microsoft Visual C++ Redistributable（x64）**  
   可使用随包提供的 `安装程序\vc_redist.x64.exe`（或服务器已安装同等运行库）。

2. 安装 **与离线 wheel 匹配的 Python x64**  
   看 `offline_bundle\wheels` 下文件名：含 **`cp313`** 表示 **Python 3.13**，含 **`cp311`** 表示 **Python 3.11**。服务器上的 Python 主版本**必须一致**，否则离线 `pip install` 会失败。  
   `安装程序` 目录下的安装包仅作示例；若与你当前的 `cp3xx` 不一致，请在外网下载**同版本** Python 安装包放进 `安装程序` 再拷贝到内网，或重新用目标 Python 版本打 wheel。

3. 建议勾选 “Add Python to PATH”，或在后续命令中使用 Python 的完整路径。

---

## 3. 离线安装 Python 依赖（在服务器上执行一次）

打开 **cmd** 或 **PowerShell**，进入项目根目录（示例 `E:\paddleOcr`）：

```bat
cd /d E:\paddleOcr
python -m venv .venv
call .venv\Scripts\activate.bat
```

先安装 Paddle（CPU），再按锁定清单安装其余依赖：

```bat
pip install --no-index --find-links offline_bundle\paddle paddlepaddle==3.3.1
pip install --no-index --find-links offline_bundle\wheels -r offline_bundle\wheels\requirements-lock.txt
```

若 `requirements-lock.txt` 中版本与你的 `paddle` 目录不一致，以**你们外网打包时锁定的版本**为准，或重新在外网生成 `requirements-lock.txt` 后再拷贝。

验证：

```bat
python -c "import fastapi, paddle; print('ok')"
```

---

### 3.1 推荐：使用脚本一键离线安装（PowerShell）
项目根目录已提供脚本：`install_offline_deps.ps1`（只使用本仓库的 `offline_bundle` 离线包，不需要外网）。

用法（PowerShell）：
```powershell
cd /d E:\paddleOcr
.\install_offline_deps.ps1
```

如服务器上 `python` 不在 `PATH`，可显式指定：
```powershell
cd /d E:\paddleOcr
.\install_offline_deps.ps1 -PythonExe 'C:\Python313\python.exe'
```

脚本会自动创建（若不存在）`.venv/`，并按顺序安装：
1) `offline_bundle\paddle\paddlepaddle-*.whl`（CPU）
2) `offline_bundle\wheels\requirements-lock.txt`（其余依赖）
3) 验证导入：`fastapi` / `paddleocr` / `paddlex`

## 4. 启动微服务

### 4.1 手动启动（测试）

在项目根目录双击或在命令行执行：

```bat
cd /d E:\paddleOcr
startup.bat
```

可选环境变量（在**同一**命令行窗口中先 `set` 再运行 `startup.bat`）：

| 变量 | 含义 | 示例 |
|------|------|------|
| `OCR_WORKERS` | uvicorn 工作进程数 | `set OCR_WORKERS=4` |
| `OCR_HOST` | 监听地址 | 默认 `0.0.0.0` |
| `OCR_PORT` | 监听端口 | 默认 `8000` |
| `PADDLE_PDX_DISABLE_MODEL_SOURCE_CHECK` | 设为 `1` 可避免内网下联网检查卡住（`startup.bat` 默认已设） | 一般无需改 |

健康检查：

- 浏览器或 `curl` 访问：`http://127.0.0.1:8000/health`  
- 期望返回：`{"success":true,"status":"ok"}`

业务接口示例：

- `POST http://服务器IP:8000/v1/ocr/document/idcard`（JSON：`imageBase64` 等，见 `README.md` / C# 示例）

### 4.2 模型路径

默认使用项目内相对路径（无需配置）：

- `offline_bundle\models\PP-OCRv4_mobile_det_infer`
- `offline_bundle\models\PP-OCRv4_mobile_rec_infer`

若模型搬到其他盘符，可设置环境变量 `OCR_DET_MODEL_DIR` / `OCR_REC_MODEL_DIR` 指向**绝对路径**后再运行 `startup.bat`。

## 5. 配置开机自动启动（任务计划程序）

推荐用 Windows **任务计划程序** 在系统启动后自动运行 `startup.bat`。

### 5.1 基本步骤

1. 打开 **任务计划程序** → **创建任务**（不要用“创建基本任务”，以便勾选最高权限）。
2. **常规**：
   - 名称：`OCRMicroservice`（可自定义）
   - 勾选 **不管用户是否登录都要运行**（视安全策略二选一）
   - 勾选 **使用最高权限运行**
3. **触发器** → 新建 → **启动时**。
4. **操作** → 新建：
   - **程序或脚本**：`cmd.exe`
   - **添加参数**：`/c "E:\paddleOcr\startup.bat"`
   - **起始于**：`E:\paddleOcr`（与项目根目录一致）
5. **条件**：按需取消“仅交流电源”（笔记本场景）。
6. **设置**：可勾选 **如果任务失败，重新启动**（间隔如 1 分钟，尝试 3 次）。

### 5.2 验证

重启服务器 → 等待数十秒 → 访问 `http://127.0.0.1:8000/health`。

---

## 6. 与 IIS 的关系

本微服务是**独立 HTTP 进程**（默认 `8000` 端口）。IIS 上的业务站点可通过 **反向代理**（ARR + URL Rewrite）将某路径转发到 `http://127.0.0.1:8000`，或由 C# 后端**直接 HTTP 调用**本机 `127.0.0.1:8000`（见 `CSharp-IIS-调用示例.md`）。

## 7. 常见问题

1. **启动很慢或卡住**  
   - 确认已设置 `PADDLE_PDX_DISABLE_MODEL_SOURCE_CHECK=1`（`startup.bat` 已默认设置）。

2. **`pip install` 报找不到包**  
   - 确认 `--find-links` 指向 `offline_bundle\wheels` 与 `offline_bundle\paddle`，且 Python 版本与 wheel 的 `cp3xx` 一致。

3. **端口被占用**  
   - `set OCR_PORT=8080` 后重新执行 `startup.bat`，并同步修改 IIS / C# 中的调用地址。

4. **任务计划运行失败**  
   - 检查“起始于”是否为项目根目录；运行账户是否有权读取 `offline_bundle`；`.venv` 是否已在该机器上创建并完成 `pip install`。
